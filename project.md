# Спецификация проекта: SecurePass Gen (v1.0)

## 1. Введение
**Название проекта:** SecurePass Gen  
**Тип приложения:** Одностраничное веб-приложение (SPA) / Static Site  
**Основная функция:** Генерация криптографически стойких паролей на стороне клиента.  
**Целевая аудитория:** Пользователи, заботящиеся о цифровой безопасности, нуждающиеся в быстрых и надежных паролях.  
**Языки интерфейса:** Русский (RU), Английский (EN).

## 2. Технологический стек
| Компонент | Технология | Примечание |
| :--- | :--- | :--- |
| **Фреймворк** | SvelteKit | Последняя стабильная версия |

| **Хостинг** | Netlify | Статический адаптер (`adapter-static`) |
| **Иконки** | Flowbite Icons / Emoji | Логотип — emoji, иконки — SVG |

## 3. Функциональные требования

### 3.1. Логика генерации пароля
1.  **Алгоритм:** Использование `crypto.getRandomValues()` для криптографической стойкости. Запрет на использование `Math.random()`.
2.  **Длина пароля:**
    *   Диапазон: от 4 до 128 символов.
    *   Значение по умолчанию: 16 символов.
3.  **Наборы символов (Чекбоксы):**
    *   Заглавные (A-Z).
    *   Строчные (a-z).
    *   Цифры (0-9).
    *   Спецсимволы (!@#$%^&*...).
    *   *Правило:* Если выбрано более 1 категории, минимальная длина пароля автоматически корректируется, чтобы быть не меньше количества выбранных категорий (для гарантии наличия символов).
4.  **Исключение неоднозначных символов:**
    *   **По умолчанию: ВКЛЮЧЕНО.**
    *   Исключаемые символы: `l`, `1`, `I`, `O`, `0`, `o`.
    *   Пользователь может отключить эту опцию вручную.
5.  **Гарантия категорий:**
    *   Алгоритм должен гарантировать наличие **хотя бы одного символа** из каждой **активной** категории в финальном пароле.
    *   *Пример:* Если выбраны "Цифры" и "Буквы", пароль не может состоять только из букв.
6.  **История:** Не сохраняется ( Stateless ).

### 3.2. Индикатор надежности
*   **Визуализация:** Прогресс-бар + текстовое описание.
*   **Уровни:**
    1.  🔴 Слабый (Мало символов, 1 категория).
    2.  🟠 Средний (Достаточная длина, 2 категории).
    3.  🟢 Надежный (Длина > 12, 3 категории).
    4.  🟣 Отличный (Длина > 16, все категории, без неоднозначных).
*   **Расчет:** На основе энтропии (длина × разнообразие символов).

### 3.3. Интерфейс и взаимодействие
1.  **Поле вывода пароля:**
    *   Read-only input.
    *   Кнопка "Копировать" (внутри или рядом с полем).
    *   Кнопка "Показать/Скрыть" (глаз).
2.  **Уведомления:**
    *   Toast-уведомление (Flowbite Toast) при копировании: "Пароль скопирован!" / "Password copied!".
3.  **Переключатель языка:**
    *   Выпадающий список или кнопки флагов (RU | EN) в шапке.
4.  **Тема:**
    *   Переключатель Светлая / Темная тема (Sun/Moon icon).
    *   Сохранение выбора в `localStorage`.

## 4. Дизайн и UI (Flowbite + Tailwind)

### 4.1. Цветовая схема
*   **Основной цвет (Primary):** Purple (Фиолетовый).
*   **Настройка Tailwind:** Расширить конфиг `tailwind.config.js` для использования оттенков `purple-500`, `purple-600` как основных для кнопок и акцентов.
*   **Фон:** Светлый (gray-50) / Темный (gray-900).

### 4.2. Компоненты
*   **Header:** Логотип (Emoji 🔒 или 🔑), Название, Переключатель языка, Переключатель темы.
*   **Main Card:** Центрированная карточка (max-w-md) с тенью.
    *   Поле пароля.
    *   Индикатор надежности.
    *   Настройки (Слайдер длины, Чекбоксы опций).
    *   Кнопка "Сгенерировать" (Большая, фиолетовая).
*   **Footer:** Копирайт, ссылка на "О проекте" (модалка или простая секция).

### 4.3. Адаптивность
*   Mobile First подход.
*   На мобильных устройствах карточка занимает всю ширину с отступами.
*   На десктопе — центрирована по центру экрана.

## 5. Техническая реализация

### 5.1. Структура проекта
```bash
src/

│   ├── i18n/             # Конфигурация svelte-i18n

│   │   └── config.ts
│   ├── utils/

```

### 5.2. Интернационализация (i18n)
*   Использование библиотеки `svelte-i18n`.
*   Файлы переводов (`src/lib/i18n/locales`):
    *   `ru.json`: { "title": "Генератор паролей", "copy": "Копировать", ... }
    *   `en.json`: { "title": "Password Generator", "copy": "Copy", ... }
*   Определение языка браузера при первом заходе.

### 5.3. Безопасность (Security Headers)
*   Настройка заголовков для Netlify (`netlify.toml` или `_headers`):
    *   `Content-Security-Policy`: `default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';` (Учитывая специфику SvelteKit/Tailwind).
    *   `X-Content-Type-Options`: `nosniff`
    *   `X-Frame-Options`: `DENY`

### 5.4. Сборка и Деплой
*   **Адаптер:** `@sveltejs/adapter-static`.
*   **Настройки `svelte.config.js`:**
    ```javascript
    import adapter from '@sveltejs/adapter-static';
    export default {
      kit: {
        adapter: adapter({
          pages: 'build',
          assets: 'build',
          fallback: 'index.html', // Для SPA роутинга
          precompress: false,
          strict: true
        }),
        prerender: {
          handleHttpError: 'warn'
        }
      }
    };
    ```
*   **Netlify:** Подключение репозитория GitHub. Build command: `npm run build`. Publish directory: `build`.

## 7. Критерии приемки (Acceptance Criteria)
1.  [ ] Сайт загружается менее чем за 2 секунды (3G simulation).
2.  [ ] При генерации пароль всегда содержит символы из выбранных категорий.

4.  [ ] Переключение языка RU/EN меняет весь текст интерфейса мгновенно.
5.  [ ] Кнопка копирования работает на мобильных и десктопах.
6.  [ ] Темная тема сохраняется после перезагрузки страницы.
7.  [ ] Код загружен на GitHub, сайт доступен по ссылке Netlify.

### Рекомендации по реализации (для разработчика)

1.  **Логика гарантии символов:**
    Не просто генерируйте случайную строку. Алгоритм должен быть таким:
    *   Создать массивы для каждой выбранной категории.
    *   Взять по 1 случайному символу из каждого выбранного массива.
    *   Заполнить оставшуюся длину случайными символами из объединения всех выбранных массивов.
    *   Перемешать итоговый массив символов (Fisher-Yates shuffle).
2.  **Tailwind Config:**
    Чтобы сделать фиолетовый цвет основным, в `tailwind.config.js` можно переопределить `primary`:
    ```javascript
    theme: {
      extend: {
        colors: {
          primary: {
            50: '#f5f3ff',
            // ... оттенки ...
            600: '#7c3aed', // Основной фиолетовый
            700: '#6d28d9',
          }
        }
      }
    }
    ```
    И использовать в компонентах Flowbite параметр `color="primary"`.
3.  **Clipboard API:**
    Используйте `navigator.clipboard.writeText()`. Не забудьте обработать ошибку, если браузер заблокирует доступ (например, без действия пользователя).
